Overview

Goals

The goals of this assignment are to:

  • Understand and implement a doubly-linked chain of nodes.
  • Gain experience working with Java references (i.e., pointers).
  • Implement an iterator.
  • Implement an exception.
  • Gain experience writing classes which implement Java interfaces.
  • Develop an application that processes input commands.
  • Get more practice with console and file I/O.

Description

For this assignment, you'll implement a LinkedList class that uses a chain of linked nodes as its data structure. You'll then implement an application that uses LinkedLists to represent a log of robot "commands", which we'll call the "Robot Log". The "commands" will be represented using Java Strings as specified below. Your application (i.e., the main class) will prompt the user to choose an operation from a set of options (described below) that manipulate the Robot Log.

It is typical for today's programmers to use ADTs that are already implemented as part of a class library (e.g., C++ Standard Template Library) or an API (e.g., Java's Application Program Interface). Understanding ADT implementations is still important to be able to make informed choices as to which ADT implementation is best suited for an application in terms of memory and time efficiencies. For example, knowing the tradeoffs between array and linked implementations of the List ADT helps us choose between Java's ArrayList and LinkedList classes. It is also important to be able to implement your own data structures when pre-built ones are not available or when standard ADTs (and their operations) are unsuitable for the intended application.

We'll provide an interface for a LinkedListADT, which you'll implement as the LinkedList class using a doubly-linked chain of nodes. You'll also code iterator and exception classes needed to support your LinkedList implementation.

Specifications

Robot Log Text File

Your program will have the ability to load in a log from a text file, and save a log to a text file. You won't be required to write any robot "commands" yourself, and you can assume that any input file provided will have properly formatted "commands". An input file will be formatted such that each line is an individual robot command, followed by a blank line at the very end of the file. There is a sample input file here for you to use when testing your code. Each line will be stored in your Robot Log as a String.

Note: You are not required to do any text parsing for this assignment. Your code will read a file line by line, but we'll assume the lines are in the correct format for robot commands. We'll simply treat these commands as strings, and we'll not be dividing those strings into smaller parts.

Information about Java I/O is available in both a summary version and a detailed version. Examples of file input are linked to in those documents.

The LinkedListADT<E> interface

A LinkedListADT<E> interface is provided for you (see LinkedListADT.java). LinkedListADT<E> is a generic interface that your LinkedList class will implement. The interface specifies the following methods, which you will implement in your LinkedList class.

Method Description
void add(E item) Adds the given item to the end of the list.
void add(int pos, E item) Adds the given item to the specified position in the list. Throws an InvalidListPositionException if the position is invalid.
E remove(int pos) Removes the item at the specified position in the list. Returns the item that was removed. Throws an InvalidListPositionException if the position is invalid.
E get(int pos) Returns the item at the given position in the list. Throws an InvalidListPositionException if the position is invalid.
boolean isEmpty() Returns whether or not the list is empty.
int size() Returns how many nodes are in the list.
Iterator<E> iterator() Returns a LinkedListIterator<E> to the list.

You may not modify the LinkedListADT interface in any way.

The ListNode<E> class

The ListNode<E> class represents one node of a chain of nodes used by your LinkedList class. This class is also provided for you; see ListNode.java. You should not modify this class. It has the following methods and constructor:

Constructor Description
ListNode<E>() Constructs an empty ListNode<E> with no data and null pointers to its previous and next nodes.
ListNode<E>(ListNode<E> prev, E data, ListNode<E> next) Constructs an empty ListNode<E> with the specified data and pointers to the specified previous and next nodes.
Method Description
E getData() Returns the data stored in this node.
ListNode<E> getPrev() Returns the ListNode<E> that comes before this ListNode<E>. Returns null if this node does not have a previous node.
ListNode<E> getNext() Returns the ListNode<E> that comes after this ListNode<E>. Returns null if this node does not have a next node.
void setData(E data) Sets the data stored in this node to the specified item.
void setPrev(ListNode<E> prev) Sets the previous node to point to the given ListNode<E>.
void setNext(ListNode<E> prev) Sets the next node to point to the given ListNode<E>.

The LinkedList<E> class

You'll code a LinkedList<E> class that your application will use to store the Robot Log. This class must implement the LinkedListADT<E> interface, which requires you to implement each method listed in the LinkedListADT<E> interface. You may add other methods as necessary; however, any additional methods other than a constructor should be private.

The LinkedListIterator<E> class

You'll code a LinkedListIterator<E> class for your LinkedList class. This class must implement Java's Iterator<E>, which requires you to implement each of these methods: hasNext(), next() and remove(). Note for remove(), you only need to throw Java's UnsupportedOperationException as described in lecture.

The InvalidListPositionException class

You'll code an exception class to be used by your LinkedList class. It must be a checked exception. You may write any constructors and methods you deem necessary for this class.

The RobotLogMain class

The application program, RobotLogMain, creates and uses an LinkedList<E> to represent and process a log of robot "commands". Similarly to Program 1, this will be a relatively simple application class. You'll write code for each of the options listed below. Your code should make appropriate use of the LinkedList, LinkedListIterator, and InvalidListPositionException classes you have written.

To help you with this, we're provideing the RobotLogMain.java file, which contains the outline of the RobotLogMain class. Download this file and use it as the starting point for your RobotLogMain implementation.

Note: Unlike Program 1, you will not read from a file given as a command-line argument. You'll instead load information from a file when given the 'l' command, described below.

The command options are:

Command format Description
l fileName Load the robot log from the given file. See the file description of the file format. You'll write code to read in from the file line by line until the end of the file (a blank line) is reached. Each of these lines should be stored as one item in your robot log. If the file cannot be found, display "Cannot find the specified file." Once the file is loaded, print each line according to the specifications under command 'p'.
p Print the robot log. The log should be printed line by line, with each line preceded by its line number. For example: 1 Name:Robot1 Move:Forward Distance:10 Speed:Fast
2 Name:Robot2 Move:Right Distance:2 Speed:Slow
3 Name:Robot1 Move:Right Distance:7 Speed:Fast
4 Name:Robot2 Move:Back Distance:8 Speed:Middle

Note that the lines are indexed starting at 1. You should not have a line # 0. If the robot log is empty, print "Empty".
r line# Remove the specified line number from the list. If an InvalidListPositionException is thrown, catch that exceptinon and print, "Invalid line number." If the argument is not an int, print, "Please enter a valid integer." Once the line has been removed, print "Successfully removed line line#".
s fileName Save the file to the specified fileName. This file should be saved in the same format as an input file; see file description. Do not save line numbers in the output file and make sure the file ends with a blank line. If the specified file cannot be written to, print, "Cannot write to the specified file." If the robot log is currently empty, print, "Cannot write to file, robot log is empty." Once the file is saved, print "Successfully saved".
t line#1 line#2 Takes a copy of the line line#1 and inserts it at line#2, shifting all commands after and including line#2 down the list. For example, given the list: 1 ABCD
2 EFGH
3 IJKL
4 MNOP
 the command "t 2 4" would produce: 1 ABCD
2 EFGH
3 IJKL
4 EFGH
5 MNOP
 If an InvalidListPositionException is thrown, catch it and print, "Invalid line number." If either argument is not an integer, print, "Please enter two valid integers." Once the line is copied, print "Successfully copied".
u line# Updates the log by coping the line at line# to the end of the log. This does not remove the line at line#. If an InvalidListPositionException is thrown, catch it and print, "Invalid line number." If the argument is not an integer, print, "Please enter a valid integer." Once the line is copied, print "Successfully copied".
