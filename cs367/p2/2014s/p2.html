Overview

Goals

The goals of this assignment are to:

  • Understand and implement a circular doubly-linked list.
  • Gain experience working with Java references (i.e., pointers).
  • Implement an iterator.
  • Implement exceptions.
  • Gain experience writing classes which implement Java interfaces.
  • Develop an application that processes input commands.
  • Get more practice with command-line arguments and file I/O.

Description

You probably have seen an LED display that loops through a sequence of letters (such as is depicted here). For this assignment you will implement a simple console-based display editor that allows a user to create, edit, save, and load sequences of characters, which can be displayed in a dot matrix form on the output console.

It is typical for today's programmers to use ADTs that are already implemented as part of a class library (e.g., C++ Standard Template Library) or an API (e.g., Java's Application Program Interface). Understanding ADT implementations is still important to be able to make informed choices as to which ADT implementation is best suited for an application in terms of memory and time efficiencies. For example, knowing the tradeoffs between array and linked implementations of the List ADT helps us choose between Java's ArrayList and LinkedList classes. It is also important to be able to implement your own data structures when pre-built ones are not available or when standard ADTs (and their operations) are unsuitable for the intended application.

In this assignment we provide you with an interface for a specialized ADT called a Loop. In addition to the main display editor application class, you will write a class which implements a Loop ADT using a circular doubly-linked list, as well as iterator and exception classes needed to support your Loop ADT implementation.

The Loop ADT

A Loop is essentially a circular List, however, unlike a List, a Loop does not have a beginning or an end and items within the Loop cannot be accessed using a position. Instead, the Loop has a current item and the ability to move forward or backwards. A Loop can be modified by removing the current item or by adding an item either before or after the current item. The Loop ADT is represented in Java by the LoopADT interface which is provided for you (see LoopADT.java). The documentation for the LoopADT interface can be found here (LoopADT Documentation ).

The MessageLoop Class

You will write a generic MessageLoop class (in a file named MessageLoop.java) which implements the LoopADT interface. In addition to the methods given in the LoopADT interface, your MessageLoop class must have a constructor that takes no arguments and creates an empty message loop. You may not add any other public constructors or methods. You may include other private fields that you find helpful, but no public fields.

The internal data structure used by the MessageLoop to contain the message must be a circular, doubly-linked list of nodes. You may not use Java's LinkedList class for this assignment. You must implement your own chain of doubly-linked nodes.

To implement the circular, doubly-linked list, you will need to define a generic DblListnode class (in a file named DblListnode.java). Your MessageLoop class will have a field of type DblListnode.

Your application will use a message loop to store the message made up of characters in their dot matrix representation, which will be stored as a List of Strings. Use Java's ArrayList class for storing the List of Strings.

Additional Classes

There are three other classes that you will need to write to go along with your MessageLoop class. You are to use exception handling to signal to the application when the following problems occur. You will also use an iterator class to access the message loop.

The EmptyLoopException class

The EmptyLoopException must be an unchecked exception. It is thrown by some methods (as described in the LoopADT interface documentation).

The UnrecognizedCharacterException class

The UnrecognizedCharacterException must be an unchecked exception. It is thrown if the input for the commands (add, insert and replace) contains a character, which isn't specified in the alphabets.txt file, which is used for converting the characters to it's corresponding dot matrix representation. Your DisplayEditor class must be able to handle this exception and help the program to continue with it's normal execution.

The MessageLoopIterator class

The LoopADT interface includes an iterator() method that returns an Iterator object. You must write a specialized iterator class, named MessageLoopIterator, which will be used as the type of iterator returned by the MessageLoop's iterator() method. The MessageLoopIterator class must implement the Iterator interface (you do not need to implement the remove method - just throw an UnsupportedOperationException). You will need to write a constructor for your MessageLoopIterator; a package-access constructor that takes a DblListnode as a parameter is suggested.

Dot Matrix Representation of Characters

Structure of a DblListnode as used by the Application

The message loop application will use the generic double linked listnodes to store the dot matrix representation of a character, using a list of strings. A pictorial representation of a single node in the doubly linked list is shown here.

DotMatrix.java

The class for converting a character to it's corresponding dot matrix representation is provided here (DotMatrix.java). This class reads in the the alphabets.txt file which has the valid characters in the first line and the dot-matrix representation of the characters below that. The order of the characters in the first line of the file must match exactly with the order in which the characters are written below in it's dot matrix representation. eg. The character A is the third character in the first line and A's dot-matrix representation is the third one. Each dot-matrix representation must be separated by a string of pound sign(#) as shown in the file. There shouldn't be any empty lines in the alphabets.txt file. The dot-matrix representation of the space character is hard-coded in the DotMatrix.java file.

The method loadAlphabets() loads the mapping between a character and it's dot-matrix representation in a TreeMap. Once loaded, you can use the getDotMatrix() function to get the corresponding dot matrix representation of a particular character.

The method getDotMatrix() takes in a string of length one (i.e. a character represented as a String) as it's argument( eg."A" or "a") and returns the character's dot matrix representation as a List of Strings. This dot matrix representation can be used as the data part of the node in the doubly linked list.

The method isValidCharacter() takes in a string of length one (i.e. a character represented as a String) as it's argument and returns true if that character is defined in the alphabets.txt file, else returns false.

You are to use the DotMatrix class in your DisplayEditor to load the mapping of character to dot-matrix representation using the alphabets.txt file, to retrieve a character's dot matrix representation and to check if a character is a valid character.

Do not modify the DotMatrix.java file.

Testing

Testing interactive programs can be frustrating if you aren't careful about recording the commands that you've entered. A bug can suddenly occur and you may not recall the exact sequence of commands to reproduce the bug. A technique for testing interactive programs is to process a fixed set of commands from an input file. Input files can be developed to test particular aspects of the program and these tests are easily repeated by running the program again on a particular input file.

Your program may be provided with two, optional, command-line arguments: the first command line argument specifies the name of an input file containing browser commands and the second argument specifies the name of the file which stores the dot matrix representation of the characters.

First Command Line Argument
If the file specified as the first command line argument doesn't exist or is unable to be opened, display "problem with input file" and quit.

Otherwise, you may assume the file contains lines that are either one of the following:

  • a single character
  • a single character followed by one space and then another character
  • a single character followed by one space and then an integer
  • a single character followed by one space and then a string of characters(including the space character)

You may also assume that the last line always has a single "q" for quit.

Second Command Line Argument

The second command line argument will allow you to play around with new font styles or characters that aren't yet in the alphabets.txt file. For your assignment purposes, we have provided you with a sample file alphabets.txt that contains the dot matrix representation of all English alphabets.

If one or more than two command-line arguments are given, display "invalid command-line arguments" and quit.

If no command-line argument is provided, your program should first prompt "Enter the dot-matrix alphabets file:". You need to enter the name of the dot matrix representation file (you could use the alphabets.txt file that we have provided and add additional characters to it). But remember that it should follow the same file format. Then the program operates as an interactive console application reading commands from the console keyboard and displaying the results in the console window.

Input from either the console or a file can be accomplished easily. If no command-line arguments are provided, construct a Scanner from System.in as shown (see also Java I/O):
