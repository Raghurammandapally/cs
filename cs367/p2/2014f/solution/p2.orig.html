Overview

Goals

The goals of this assignment are to:

  • Understand and implement a doubly-linked chain of nodes.
  • Gain experience working with Java references (i.e., pointers).
  • Implement exceptions.
  • Gain experience writing classes which implement Java interfaces.
  • Develop an application that processes input commands.
  • Get more practice with console and file I/O.

Description

Twitter is a popular social media platform which allows user to post short messages ("Tweets") to their profile. These messages must be less than 140 characters. A user on Twitter has a list of followers whose tweets they want to track. Twitter takes this list of followers and creates a timeline of the all the follower tweets in a single,chronologically sorted list, called the timeline.

For this assignment, you will write a program that simulates the basic user experience of Twitter using a doubly-linked chain of nodes. A set of files, each representing a user's tweets, will be passed from the command line. These files will be processed and loaded into the program. From these users you will generate a timeline represented by an ordered list of tweet objects, which you will then be able to perform operations (described below) on.

You'll write a main class as your application as was done in Program 1. Additionally, you'll write a LinkedList class that is used by your application to hold lists of tweets from individual users as well as a Timeline list made up of multiple user's tweets. Your main class prompts the user to choose an operation that manipulates the Timeline.

Your LinkedList class must implement the ListADT interface using a doubly-linked chain of nodes. You'll also implement an exception class used by your LinkedList to signal problems.

Specifications

The ListADT<E> interface

A ListADT<E> interface is provided for you (see ListADT.java). The ListADT<E> interface is a generic interface that you will implement in your implementation of a LinkedList. The interface describes the following methods, which you will implement in your LinkedList class.

Method Description
void add(E item) Add item to the end of the List.
void add(int pos,E item) Add item at position pos in the List, moving the items originally in positions pos through size() - 1 one place to the right to make room.
E remove(int pos) Removes the item at the position in the list. Returns the item that was removed.
E get(int offset) Return the item at position pos in the List.
boolean isEmpty() Returns whether or not the list is empty.
int size() Returns how many nodes are in the list.

Do not modify the ListADT interface in any way.

The ListNode<E> class

The ListNode<E> class represents one node of a LinkedList. This class is also provided for you (see ListNode.java). It has the following methods and constructor:

Constructor Description
ListNode<E>(E data) Constructs a ListNode<E> with the specified data and a null pointer to its previous and next nodes.
ListNode<E>(E data,ListNode<E> prev, ListNode<E> next) Constructs a ListNode<E> with the specified data and a pointer to the specified prev and next nodes.
Method Description
E getData() Returns the data stored in this node.
ListNode<E> getNext() Returns the ListNode<E> that comes after this ListNode<E>. Returns null if this node does not have a next node.
ListNode<E> getPrevious() Returns the ListNode<E> that comes before this ListNode<E>. Returns null if this node does not have a next node.
void setData(E data) Sets the data stored in this node to the specified item.
void setNext(ListNode<E> prev) Sets the next node to point to the given ListNode<E>.
void setPrevious(ListNode<E> prev) Sets the previous node to point to the given ListNode<E>.

Do not modify the ListNode class in any way.

The Tweet class

The Tweet class represents a single Tweet from a user. A shell of this class is also provided for you (see Tweet.java). It has the following methods and constructor:

Constructor Description
Tweet(int Time,String message,String User) Constructs a Tweet with the specified message at a given time. Throws a TweetTooLongException if message over 140 characters.
Method Description
String getMessage() Returns the stored message of the Tweet.
int getTime() Get the time a Tweet was tweeted
String getUser() Returns the user who tweeted the Tweet.
void print() Print the Tweet with the following format: <TIME> <USER>:<MESSAGE>.

The TweetTooLongException class

You will write an exception class to be used by your implementation of a Tweet to signal when a Tweet which is too long is trying to be created. This class should be a checked exception. You may write any constructors and methods you deem necessary for this class. Your main Twitter program should catch any lines which are too long and not add them to the list of user tweets.

The Timeline class

The Timeline class should use your LinkedList class to build a time ordered list of follower tweets. Tweets with lower Time fields should come earlier in the list. A shell of this class is also provided for you (see Timeline.java), you must implement all the methods. It has the following methods and constructor:

Constructor Description
Tweet() Constructs an empty Timeline
Method Description
void add(Tweet tweet) Adds a single tweet to the timeline
void add(List tweets) Adds an ordered list of tweets to the timeline
void remove(String user) Removes all tweets by user from the timeline.
void print() Print each tweet in the timeline on it's own line
void print(int time) Print each tweet since time in the timeline on it's own line
Timeline search(String keyword) Create a new Timeline containing only tweets containing keyword

The Twitter class

The application program, Twitter, reads in each file from the command line. Twitter should then read in each file, creating a list of followers and an associated list of Tweets. Twitter then should create and use a Timeline object to represent and simulate the Timeline. Assume initially all user files passed at the command line are followed. You will write code for each of the options listed below. Your code should make use of the LinkedList and ElementNotFoundException classes you have written, where appropriate.

The Twitter.java file contains the outline of the Twitter class. Download this file and use it as the starting point for your Twitter implementation.

For this program, you'll access some number of files USER.txt containing one tweet per line, with the format <TIME>:<MESSAGE> on each line. You should assume the prefix of each file name is the name of the user. The pathname of these files will be provided by using command-line arguments. For this assignment, assume that time values are positive integers, representing the number of seconds since this tweet was generated; in this case older tweets have higher integer values. For simplicity, assume that no two tweets occur at the same time. User text files will already be ordered with the most recent tweet being the first in the file. All user input which does not correspond to a command below should print the error message "Invalid Command".

The command options are:

Command format Description
list users Display a list of all users, one per line. Users should be displayed in same order as their files were entered on the command line.
list following Display a list of the users you follow, one per line. should be displayed in same order as their files were entered on the command line.
follow user Adds user to your list of followers and update your timeline. If the user is already in your list of followers, print "Warning: User already followed". Check that user is valid.
unfollow user Removes user to your list of followers and update your timeline. If the user is not in your list of followers, print "Warning: User not followed". Check that user is valid.
search string Generate and print search a new list of all tweets in your timeline which contain this string.
print Print your timeline, one tweet per line.
print int Print your timeline for all tweets occuring since int, one tweet per line.
quit Exit your program
